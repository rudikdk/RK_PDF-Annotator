RK PDF-Annotator

RK PDF-Annotator is a Windows Excel add-in for annotating P&ID PDF documents from the currently open Excel workbook.

Basic Workflow

  1. Open the workbook that contains your component or tag list.
  2. Select the worksheet with the tag data.
  3. Click PDF Annotator -> Annotate PDFs.
  4. Confirm the header row and choose the tag column.
  5. Confirm the tag format, including minimum parts, maximum parts, and separators.
  6. Select one or more note columns, such as description, location, comments, or service.
  7. Optionally enable watermarking and select one or more columns whose values should appear beneath each matching tag.
  8. Choose the PDF file and output path.
  9. Click Annotate.

Preview Tags

Use Preview Tags before annotating to confirm that the add-in is reading the expected worksheet data.

PDF Output

The add-in first builds an index of all PDF tags that match the configured tag format. It then compares that index with the selected Excel tag column. Matching words are highlighted, and a PDF note annotation is added with the selected Excel data.

When processing finishes, the add-in creates a formatted PDF Tag Report worksheet in the active workbook. The report includes a summary, duplicate Excel tags, found tags, Excel tags not found in the PDF, and per-page tag counts. A CSV copy is also written next to the output PDF as a file-based fallback.

Tag Format

Use Min parts, Max parts, and Separators to describe how tags look in the PDF. The default matches 3-5 part tags separated by - or ..

Watermarks

Enable Watermark to draw selected Excel values directly beneath every matching tag. Select one or more columns, use Move up and Move down to control their order, and adjust the font size, text color, or white background as needed.

Multiple non-empty values are placed on one line separated by / . The watermark is written natively into the output PDF and does not require the web application or Python.

Color Rules

Use the Color rules section in the Annotate PDFs dialog to control the highlight color per tag, instead of a single fixed color for every match.

  • Click Add rule to create a rule based on a tag part (e.g. the 2nd dash- or dot-separated segment of the tag) or an Excel column value.
  • Choose a match type: exact match, contains, greater than, less than, or has any value.
  • Pick the highlight color for the rule.
  • Use Move up / Move down to reorder rules - when more than one rule matches a tag, the last matching rule in the list wins.
  • Use Edit or Remove to change or delete an existing rule.
  • Set the Default highlight color for tags that do not match any rule.

Color rules use the same matching logic as the color rules feature in the PID Annotator web application, so behavior is consistent between the two tools.

Troubleshooting

  • If no tags are found, confirm that the selected PDF contains selectable text rather than only scanned images.
  • If the report sheet misses expected PDF tags, adjust the tag format part count or separators and run annotation again.
  • If the wrong data appears, confirm the active worksheet and header row before running the annotation.
  • If Excel blocks the add-in, right-click the downloaded .xll, open Properties, and choose Unblock if Windows shows that option.

Third-Party Libraries

RK PDF-Annotator is built on the following open-source libraries:

  • ExcelDna.AddIn (1.9.0) - hosts the add-in inside Excel and provides the ribbon UI. Licensed under the zlib/libpng License.
  • Newtonsoft.Json (13.0.3) - JSON parsing and serialization. Licensed under the MIT License.
  • PDFsharp (1.50.5147) - generates and merges the watermark overlay into the output PDF. Licensed under the MIT License.
  • PdfPig (UglyToad.PdfPig, 0.1.14) - extracts text from PDFs to locate matching tags. Licensed under the Apache License 2.0.
